Feature Implementation
🚶
Motion Detection
PIR sensor on interrupt-driven GPIO. Firmware debounces signal and fires IFTTT webhook on confirmed motion event.
HC-SR501 → GPIO D5 → ISR → HTTPS POST → IFTTT → Notification + relay trigger
🔥
Fire & Smoke Alert
MQ-2 read on ADC A0 every 500 ms. When raw count exceeds calibrated smoke threshold, emergency alert sent and ventilation relay triggered.
ADC read → threshold compare → if > 400 counts → alarm webhook + relay HIGH
🔒
Auto Door Locking
Relay-driven solenoid lock. Automatically engages on motion alert after 30 s timer, or triggered remotely via Google Assistant command.
Voice: "Lock the door" → Google Assistant → IFTTT → Webhook → ESP8266 → GPIO LOW → relay energised → solenoid locked
🎙️
Voice Control
Google Assistant applet on IFTTT maps spoken phrases to webhook calls. Supports fan, lights, and door lock commands without a dedicated mobile app.
Phrases: "Turn on fan", "Turn off lights", "Activate security" → IFTTT applet → relay GPIO
⚡
Energy Automation
Lights and fans automatically switch off when no motion is detected for 10 minutes. Reduces idle power consumption without manual intervention.
No PIR trigger for 600 s → idle flag set → relay OFF for non-essential loads
☁️
Cloud Telemetry
All sensor events logged to IFTTT with timestamp. Bidirectional: ESP8266 both publishes events and receives command webhooks from cloud.
ESP8266HTTPClient → POST /trigger/event/with/key/{token} → JSON payload {value1, value2}
Firmware Logic (Arduino C)
smart_home_main.ino — NodeMCU ESP8266 
// ── Smart Home Firmware — Kishore Raja K ──────────────────────────────────
#include <ESP8266WiFi.h>
#include <ESP8266HTTPClient.h>

// ── Pin definitions ────────────────────────────────────────────────────────
#define PIR_PIN       D5    // Digital GPIO — motion sensor interrupt
#define MQ2_PIN       A0    // Analog ADC — smoke/gas sensor
#define RELAY_LOCK    D1    // Relay CH1 — door lock solenoid
#define RELAY_FAN     D2    // Relay CH2 — fan control
#define SMOKE_THRESH  400   // ADC threshold (~calibrated to MQ-2 curve)
#define IDLE_TIMEOUT  600000 // 10 min no-motion → auto power-off

const char* ssid       = "HOME_WIFI";
const char* password   = "*****";
const char* iftttKey   = "YOUR_IFTTT_KEY";

volatile bool motionFlag  = false;
unsigned long lastMotion  = 0;

// ── ISR: motion detected ───────────────────────────────────────────────────
void ICACHE_RAM_ATTR onMotion() { motionFlag = true; }

void sendIFTTT(String event, String val1) {
  if (WiFi.status() != WL_CONNECTED) reconnectWiFi();
  HTTPClient http;
  String url = "http://maker.ifttt.com/trigger/" + event +
               "/with/key/" + iftttKey +
               "?value1=" + val1;
  http.begin(url);
  http.GET();   // fire-and-forget telemetry POST
  http.end();
}

void setup() {
  pinMode(PIR_PIN,    INPUT);
  pinMode(RELAY_LOCK, OUTPUT); digitalWrite(RELAY_LOCK, HIGH); // active LOW — locked
  pinMode(RELAY_FAN,  OUTPUT); digitalWrite(RELAY_FAN,  HIGH);
  attachInterrupt(digitalPinToInterrupt(PIR_PIN), onMotion, RISING);
  connectWiFi();
}

void loop() {
  // ── Motion event ──────────────────────────────────────────────────────
  if (motionFlag) {
    motionFlag  = false;
    lastMotion  = millis();
    sendIFTTT("motion_detected", "PIR_D5");
  }

  // ── Smoke / fire check ────────────────────────────────────────────────
  int smoke = analogRead(MQ2_PIN);
  if (smoke > SMOKE_THRESH) {
    sendIFTTT("fire_alert", String(smoke));
    digitalWrite(RELAY_FAN, LOW);  // turn on exhaust fan
  }

  // ── Idle energy cut-off ───────────────────────────────────────────────
  if (millis() - lastMotion > IDLE_TIMEOUT) {
    digitalWrite(RELAY_FAN,  HIGH); // all non-essential loads OFF
  }

  delay(500);
}
End-to-End Telemetry Pipeline
01
Physical Sensor Event
PIR goes HIGH on motion, or MQ-2 ADC exceeds smoke threshold. ISR sets flag in ESP8266 firmware; main loop picks it up within 500 ms.
PIR → GPIO D5 interrupt → motionFlag = true
02
Local Actuation Decision
MCU applies threshold logic immediately — no cloud latency for safety-critical actions like smoke detection. Relay fires in under 1 ms from detection.
if (smoke > 400) digitalWrite(RELAY_FAN, LOW) → fan ON
03
HTTPS Telemetry Uplink
ESP8266 fires HTTPS GET/POST to IFTTT Maker Webhook endpoint with event name and sensor values encoded as URL parameters.
GET maker.ifttt.com/trigger/motion_detected/with/key/<token>?value1=PIR_D5
04
IFTTT Applet Processing
IFTTT applet receives event, executes configured actions — push notification to phone, Google Assistant announcement, or return webhook to ESP8266.
Trigger: motion_detected → Action: notify + lock_door webhook → ESP8266
05
Voice Command Downlink
User speaks command to Google Assistant. Assistant routes to IFTTT applet, which fires webhook to ESP8266 — completing the bidirectional command loop.
"Hey Google, lock the door" → IFTTT → webhook → ESP8266 → RELAY_LOCK LOW
